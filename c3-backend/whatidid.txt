1,MongoDB Atlas setup (full walkthrough)

Go to https://www.mongodb.com/cloud/atlas/register
Sign up using the C3 club Gmail

2,Get your connection string

On the Atlas dashboard, click Connect on your cluster
Choose "Drivers"
It shows a connection string like:

mongodb+srv://c3admin:<password>@c3cluster.xxxxx.mongodb.net/?retryWrites=true&w=majority


3,Generate a JWT Secret
This is just a random long string used to "sign" your login tokens so no one can fake one. You don't need a special tool — just generate a random string yourself.
Easiest way, run this in your terminal (Node is already installed):
bashnode -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

   KEY name (e.g. " date" instead of "date") silently breaks

   console.log(req.body) in the controller temporarily to see 
   the RAW keys received, spaces and all. Removed after confirming.

4. JWT expiry changed: 7d -> 180d (6 months)
   - In authController.js generateToken()
   - IMPORTANT GOTCHA: jsonwebtoken's expiresIn uses 'd' for 
     days, NOT 'm' for months -- 'm' means MINUTES in this 
     library. Always write '180d', never '6m'. Left a comment 
     in the code as a warning for future devs.
   - Only affects tokens issued AFTER the change -- old tokens 
     in Postman still expire on their original schedule
